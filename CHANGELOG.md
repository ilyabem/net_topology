# Changelog

All notable changes to this project will be documented in this file.

## [Unreleased]

## [1.3.0] - 2026-05-25
### Added
- **Мобильные устройства** (`mobile_device`) — новый тип для Android/iOS устройств
  с рандомизированными MAC-адресами (LAA). Windows 11 с LAA + TTL=128 корректно
  определяется как `windows_endpoint`, а не мобильное.
- **Hostname-эвристика** — определение ОС по имени хоста: `msi-*`, `ubuntu-*`,
  `debian-*`, `DESKTOP-*`, `iphone`, `galaxy` и другие паттерны.
- **Адаптивный layout** — размер canvas масштабируется под число узлов
  (`max(800, 90×√n)` пикселей). При 18 узлах минимальное расстояние ≥ 96px.
- **`_spread_overlapping()`** — алгоритм принудительного разведения узлов,
  исключает наложение иконок при плотных топологиях.

### Fixed
- Наложение узлов друг на друга при большом количестве устройств в сети.
- Устройства с рандомизированными MAC определялись как `unknown` — теперь `mobile_device`.
- Сканирующий хост (без MAC в ARP) определяется по hostname.

## [1.2.0] - 2026-05-24
### Added
- **OUI-база производителей** (`oui_db.py`) — локальный кэш ~39 000 записей.
  Источники: Wireshark manuf + IEEE OUI CSV. Авто-обновление раз в 60 дней.
- **`--update-oui`** — CLI-флаг для ручного обновления базы.
- **Кнопка обновления OUI** в GUI (Правка → 🔄 Обновить OUI-базу...).
- **VoIP-телефоны** (`voip_phone`) — новый тип, голубой шестиугольник.
  Определяются по OUI (Grandstream, Yealink, Polycom, Snom, Fanvil...).
- **Gateway-эвристика** — IP оканчивающийся на `.1` или `.254` получает
  буст +4.5 к типу `router`.
- **Расширенные типы устройств**: `windows_server`, `linux_server`,
  `windows_endpoint`, `linux_endpoint`, `printer` вместо общих `server`/`endpoint`.

### Fixed
- `192.168.x.1` определялся как `linux_server` из-за портов 80/443 — исправлено gateway-эвристикой.
- KYOCERA Display, Pantum, Grandstream определялись неверно — добавлены в OUI-правила.
- pysnmp 7.x (lextudio) несовместим со старым sync API — добавлен автодетектор версии.

## [1.1.0] - 2026-05-22
### Added
- **Поддержка нескольких подсетей** (`--subnets 192.168.1.0/24 10.0.0.0/24`).
- **Параллельное сканирование** (`--parallel`) через `ThreadPoolExecutor`.
- **Мультиинтерфейсные устройства** — автоматическое объединение узлов
  с одинаковым MAC (`merge_by_mac()`). Золотистая двойная рамка в GUI + символ ★.
- **Ручное объединение/разделение** узлов через контекстное меню.
- **Фильтр типов устройств** — выпадающий список с чекбоксами, мгновенное
  скрытие/показ без пересканирования. Группы: сетевое / серверы / конечные точки / мобильные.
- **Сканирование в фоне** (`QThread`) — GUI не зависает во время скачивания.
- **`normalize_mac()`** — нормализация MAC из форматов Linux/Windows/Cisco.

### Fixed
- Циклический импорт `gui.py` ↔ `net_topology.py` — исправлен lazy-импортом.
- `Device.from_dict()` дублировал интерфейсы при десериализации.

## [1.0.0] - 2026-05-11
### Added
- Сканирование сети через ARP (Scapy) и ping (nmap).
- Определение типов устройств: router, switch, firewall, bridge, server, endpoint.
- Взвешенная классификация: SNMP + порты + TTL + OUI.
- Граф топологии на NetworkX со spring layout.
- GUI на PyQt5: перетаскивание узлов, контекстное меню, сохранение/загрузка JSON.
- Экспорт в JSON и GraphML.
- CLI-режим (`--no-gui`) с текстовой таблицей.
- Юнит-тесты (pytest).
